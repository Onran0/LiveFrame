# LFA

**LFA** (LiveFrame Animation) - текстовый формат
описания анимаций для скелетов **VoxelCore**, использующийся
контент-паком **LiveFrame**.

Формат старается быть похожим на **VCM** для удобства
и большего сходства с движком.

Формат описания элементов и иерархии такой же, как и в
**VCM**:

```
@элемент атрибут (значение1,значение2,...) атрибут2 "значение3" {
    @внутренний_элемент_1 ...
    @внутренний_элемент_2 ...
}
```

> [!NOTE]
> Звёздочка (`*`) слева от названия атрибута означает, что он - опционален.

## Основные элементы

### Элемент `metadata`

***Описание***

Используется для описания метаданных файла и других элементов.

> [!NOTE]
> Обязателен.

***Атрибуты***

- `version` - версия формата *LFA* файла вида `major.minor`. Актуальное значение: `1.0`;\


- `*euler-order` - порядок вращения углов Эйлера
  для всех элементов в файле.\
  Значение по-умолчанию: `xyz`.


- `*relativize-transforms` - булев, определяющий, должны ли
все представленные в клипах трансформации релятивизироваться
относительно начальной позы в `skeleton`. Настройка была создана
в частности для `VCM`, загрузчик которого вращает вершины костей
при загрузке модели. Однако для прямого риггинга это может быть
нежелательно.\
Значение по-умолчанию: `true`

***Пример***

```
@metadata version 1.0
```

### Элемент `skeleton`

***Описание***

Используется для описания скелета, трансформируемого клипами, а также для определения начальной позы.

> [!NOTE]
> Может быть определён только единожды на весь файл.

> [!WARNING]
> Все кости использующиеся в клипах обязательно должны быть определены в `@skeleton`!

***Допустимые вложенные элементы***

`skeleton#bone`.

***Пример***

```
@skeleton {
    @bone name "head" rotation (0, 0, 0)
    @bone name "left_arm" rotation (0, 0, 0) position (0, 0, 0)
    @bone name "right_arm" rotation (0, 0, 0) position (0, 0, 0)
}
```

### Под-элемент `skeleton#bone`

***Описание***

Используется для описания конкретной кости скелета, а также для её начальной позы.

> [!NOTE]
> Ввиду устройства скелетов в **VoxelCore**, в описании отсутствует иерархия, а также
> все кости должны иметь уникальное название. То-есть, описание скелета с двумя костями
> с названием, например, `head`, будет считаться невалидным.

***Атрибуты***

- `name` - имя кости;
- `*position` - начальная позиция кости в формате `(x, y, z)`. Значение по-умолчанию: `(0, 0, 0)`;
- `*rotation` - начальная позиция кости в формате `(x, y, z)` для
углов Эйлера, или `(x, y, z, w)` для кватернионов. Значение по-умолчанию:
`(0, 0, 0)` в углах Эйлера, или `(0, 0, 0, 1)` в кватернионе;
- `*scale` - начальная позиция кости в формате `(x, y, z)`. Значение по-умолчанию: `(1, 1, 1)`.

***Пример***

```
... {
    @bone name "head" rotation (0, 0, 0)
    @bone name "left_arm" rotation (0, 0, 0) position (0, 0, 0)
    @bone name "right_arm" rotation (0, 0, 0) position (0, 0, 0)
}
```

### Элемент `clip`

***Описание***

Используется для описания конкретного клипа анимации.

***Атрибуты***

- `name` - название клипа;


- `*loop` - булев, определяющий, может ли клип быть
зациклен. Используется для автоматического расчёта
касательных и контрольных кватернионов для `cubic-spline`,
`squad`.\
Значение по-умолчанию: `false`;

> [!WARNING]
> В зацикленных клипах **LFA** первый ключ не должен дублироваться
> на последний. Для задачи длительности сегмента между последним
> ключ-кадром и первым, используйте поле `duration`.


- `*duration` - длительность клипа в секундах. Если
не указано явно, то вычисляется автоматически на основе
времени ключ-кадров. Явное определение используется для
читабельности или определения времени сегмента между
последним и первым ключ-кадром в зацикленных клипах
(`segmentDuration = duration - lastKeyTime; segmentTime = duration`).

***Допустимые вложенные элементы***

`keyframe`.

***Пример***

```
@clip name "test" loop true {
    ...
}
```

### Элемент `keyframe`

***Описание***

Используется для описания ключ-кадров в анимации. В описание входит:
- позиции, вращения и масштабы костей;
- время начала ключ-кадра;
- триггеры.

***Атрибуты***

- `time` - время начала ключ-кадра в секундах.

***Допустимые вложенные элементы***

`event`;\
`scope`;\
`bone`.

***Пример***

```
... {
    @keyframe time 0 {
        ...
    }
    
    @keyframe time 1 {
        ...
    }
}
```

> [!NOTE]
> Список ключ-кадров в `animation` должен быть отсортирован в порядке возрастания времени.

### Элемент `event`

***Описание***

Используется для оповещения скриптинга о начале конкретного
ключ-кадра и опционально для передачи константного значения.

***Атрибуты***

- `name` - название триггера;
- `*value` - (опционально) значение триггера. может быть строкой, булевом, числом или одномерным
массивом, состоящего из любых предыдуще перечисленных типов.

***Пример***

```
... {
    ... {
        @event name "speed" value 2
        
        ...
    }

    ... {
        @event name "attack"
        
        ...
    }
    
    ... {
        @event name "speed" value 1
        
        ...
    }
}
```

### Элемент `bone`

***Описание***

Используется для описания трансформаций конкретной кости
на текущем ключ-кадре.

***Атрибуты***

- `name` - имя кости;


- `*interp`, `*in-interp`, `*out-interp` - идентификатор/тип интерполяции для позиции и масштаба по-умолчанию.\
Стандартные типы: `lerp`; `cubic-spline`; `step`, по-умолчанию: `lerp`;

> [!NOTE]
> Касательные (тангенсы) для `cubic-spline` вычисляются автоматически.
> Для ручной настройки смотрите документацию об элементе `interp`.

- `*rotation-interp`, `*in-rotation-interp`, `*out-rotation-interp` - идентификатор/тип интерполяции для вращения по-умолчанию.\
Стандартные типы: `nlerp`; `slerp`; `squad`; `step`, по-умолчанию: `nlerp`;

> [!NOTE]
> Контрольные кватернионы для `squad` вычисляются автоматически.
> Для ручной настройки смотрите документацию об элементе `interp`.

> [!NOTE]
> Идентификатор интерполяции означает идентификатор кастомной
> конкретизации какого-либо типа интерполяции (см. элемент `interp`).

> [!NOTE]
> Атрибуты интерполяции с приставкой `in` используются для определения
> типа интерполяции для перехода от предыдущего ключ-кадра к текущему,
> с приставкой `out` для перехода от текущего ключ-кадра к следующему.

> [!NOTE]
> При умолчании любого из атрибутов для настройки интерполяции (
> `interp`, `rotation-interp`, `in-interp`, `out-interp`, `in-rotation-interp`,
> `out-rotation-interp`), явно указанное значение будет искаться вверх по иерархии
> (т.е. от родителя к родителю родителя и т.д.), пока не найдётся явно
> указанное значение. Значение по-умолчанию будет использовано только если
> явно указанное значение не было найдено. Если же говорить кратче,
> то приоритет выбора значения такой:
> `transformation (position, rotation, scale) > bone > scope's > default`

> [!WARNING]
> Если использовать атрибуты без приставки (`interp`, `rotation-interp`),
> то указанный вид интерполяции будет применён к обоим переходам, т.е.
> от предыдущего ключ-кадра к текущему, и от текущего к следующему.

> [!WARNING]
> Если в вашей анимации будет переход от условного ключ-кадра `A` к
> ключ-кадру `B`, и при этом в ключ-кадре `A` задана интерполяция типа `out`
> (`out-interp`, `out-rotation-interp`) к какому либо элементу
> трансформации (`position`, `rotation`, `scale`) явно или неявно (например,
> через `interp`, `rotation-interp` или наследование интерполяционных атрибутов),
> и при этом в ключ-кадре `B` есть такая же ситуация, но с интерполяцией типа `in`
> (явно `in-interp`, `in-rotation-interp`, неявно `interp`, `rotation-interp`),
> то к этому переходу будет выбрана `in` интерполяция из ключ-кадра `B`.

***Допустимые вложенные элементы***

`position`;\
`rotation`;\
`scale `.

***Пример***

```
... {
    ... {
        ...
        
        @bone name "spine" {
            ...
        }
        
        @bone name "head" {
            ...
        }
    }
}
```

### Элемент `scope`

***Описание***

Используется для обобщения интерполяционных параметров костей.

***Атрибуты***

- `*interp`, `*rotation-interp`, `*in-interp`, `*out-interp`, `*in-rotation-interp`, `*out-rotation-interp`.

***Допустимые вложенные элементы***

`scope`;\
`bone `.

***Пример***

```
... {
    ... {
        @scope interp "lerp" {
            ...
            
            @scope interp "cubic-spline" rotation-interp "slerp" {
                ...
            }
        }
    }
}
```

### Элемент `position`

***Атрибуты***

- `value` - массив вида `(x, y, z)`, представляющий позицию;
- `*interp`, `*in-interp`, `*out-interp`.

***Пример***

```
... {
    ... {
        ... {
            @position value (0, 0, 0)
        }
    
        ... {
            @position value (0, 2, 2) interp "lerp"
        }
        
        ... {
            @position value (0, 4, 4) interp "cubic-spline"
        }
    }
}
```

### Элемент `rotation`

***Атрибуты***

- `value` - вращение, представленное углами Эйлера в градусах (массив вида `(x, y, z)`),
или кватернионом (массив вида `(x, y, z, w)`);
- `*rotation-interp`, `*in-rotation-interp`, `*out-rotation-interp`.

***Пример***

```
... {
    ... rotation-interp "slerp" {
        ... {
            @rotation value (0, 0, 0, 1) rotation-interp "squad"
        }
    
        ... {
            @rotation value (90, 90, 90) rotation-interp "nlerp"
        }
    }
}
```

### Элемент `scale`

***Атрибуты***

- `value` - массив вида `(x, y, z)`, представляющий масштаб;
- `*interp`, `*in-interp`, `*out-interp`.

***Пример***

```
... {
    ... {
        ... {
            @scale value (0, 0, 0)
        }
    
        ... {
            @scale value (0, 2, 2) interp "lerp"
        }
    }
}
```

## Элементы для сложной анимации

### Элемент `interp`

***Описание***

Используется для конкретизации параметров для некоторых видов интерполяций (в т.ч. `cubic-spline`, `squad`)

***Атрибуты***

- `id` - идентификатор конкретизированной интерполяции, который можно будет использовать в анимациях;
- `type` - тип конкретизируемой интерполяции. Доступные значения: `cubic-spline`; `squad`.

***Поля***

Для описания параметров различных видов интерполяций, используется под-элемент `field`,
имеющий атрибуты `name` и `value`, которые содержат имя и значение параметра.

**Поля для `cubic-spline`**

- `in-tangent` - вектор, описывающий направление и скорость изменения значения при входе в ключ-кадр.
- `out-tangent` - вектор, описывающий направление и скорость изменения значения при выходе из ключ-кадра.

***Пример***

```
@interp id "custom-interp" type "cubic-spline" {
    @field name "in-tangent" value (0,3,1)
    @field name "out-tangent" value (3,3,1)
}
```

**Поля для `squad`**

- `in-control` - вспомогательный кватернион, определяющий кривизну траектории при входе в данную точку. 
Влияет на форму сегмента перед этим ключ-кадром;

- `out-control` - вспомогательный кватернион, определяющий кривизну траектории при выходе из данной точки. 
Влияет на форму сегмента после этого ключ-кадра.

***Пример***

```
@interp id "custom-interp" type "squad" {
    @field name "in-control" value (0,0.5,0,0.5)
    @field name "out-control" value (0.25,0.25,0.25,0.25)
}
```

### Под-элемент `interp#field`

***Описание***

Используется для описания конкретного поля интерполяции.

***Атрибуты***

- `name` - название поля;
- `value` - значение поля.

***Пример***

```
... {
    @field name "in-tangent" value (0,3,1)
    @field name "out-tangent" value (3,3,1)
}
```
