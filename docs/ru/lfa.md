# LFA

**LFA** (LiveFrame Animation) - текстовый формат
описания анимаций для скелетов **VoxelCore**, использующийся
контент-паком **LiveFrame**.

Формат старается быть похожим на **VCM** для удобства
и большего сходства с движком.

Формат описания элементов и иерархии такой же, как и в
**VCM**:

```
@элемент атрибут (значение1,значение2,...) атрибут2 "значение3" {
    @внутренний_элемент_1 ...
    @внутренний_элемент_2 ...
}
```

## Основные элементы

### Элемент `animation`

***Описание***

Используется для описания конкретной анимации.

***Атрибуты***

- `name` - название анимации;

- `euler-order` - порядок вращения углов Эйлера
для всех вложенных элементов.\
Значение по-умолчанию: `xyz`.

***Допустимые вложенные элементы***

`keyframe`.

***Пример***

```
@animation name "test" euler-order "xyz" {
    ...
}
```

### Элемент `keyframe`

***Описание***

Используется для описания ключ-кадров в анимации. В описание входит:
- позиции, вращения и масштабы костей;
- время начала ключ-кадра;
- переопределение интерполяций;
- триггеры.

***Атрибуты***

- `time` - время начала ключ-кадра в секундах.

***Допустимые вложенные элементы***

`trigger`;\
`union`;\
`bone`.

***Пример***

```
... {
    @keyframe time 0 {
        ...
    }
    
    @keyframe time 1 {
        ...
    }
}
```

> [!NOTE]
> Список ключ-кадров в `animation` должен быть отсортирован в порядке возрастания времени.

### Элемент `trigger`

***Описание***

Используется для оповещения скриптинга о начале конкретного
ключ-кадра и опционально для передачи константного значения.

***Атрибуты***

- `name` - название триггера;
- `value` - значение триггера (опционально).

***Пример***

```
... {
    ... {
        @trigger name "speed" value 2
        
        ...
    }

    ... {
        @trigger name "attack"
        
        ...
    }
    
    ... {
        @trigger name "speed" value 1
        
        ...
    }
}
```

### Элемент `bone`

***Описание***

Используется для описания трансформаций конкретной кости
на текущем ключ-кадре.

***Атрибуты***

- `name` - имя кости;


- `interp`, `in-interp`, `out-interp` - идентификатор интерполяции для позиции и масштаба по-умолчанию.\
  Дефолтные значения: `lerp`; `cubic-spline`; `step`, по-умолчанию: `lerp`;

> [!NOTE]
> Тангенсы для `cubic-spline` вычисляются автоматически.
> Для ручной настройки смотрите документацию об элементе `interp`.

- `rotation-interp`, `in-rotation-interp`, `out-rotation-interp` - идентификатор интерполяции для вращения по-умолчанию.\
  Дефолтные значения: `nlerp`; `slerp`; `squad`; `step`, по-умолчанию: `nlerp`;

> [!NOTE]
> Контрольные кватернионы для `squad` вычисляются автоматически.
> Для ручной настройки смотрите документацию об элементе `interp`.

> [!NOTE]
> Атрибуты интерполяции с приставкой `in` используются для определения
> типа интерполяции для перехода от предыдущего кадра к текущему,
> с приставкой `out` для перехода от текущего кадра к следующему.

> [!NOTE]
> Атрибуты `in-interp`, `out-interp`, `interp`, `in-rotation-interp`,
> `out-rotation-interp` и `rotation-interp` могут быть переопределены
> в элементах трансформации (`position`, `rotation`, `scale`),
> однако по-умолчанию будет использовано значение кости. Также, если
> в атрибутах кости не указаны перечисленные, то если кость имеет родительский
> `union`, то будут использоваться значения из него. Если же и в нём
> не указаны, или его нет, то будут использованы значения по умолчанию.

> [!WARNING]
> Если использовать атрибуты без приставки (`interp`, `rotation-interp`),
> то указанный вид интерполяции будет применён к обоим переходам, т.е.
> от предыдущего ключ-кадра к текущему, и от текущего к следующему.

> [!WARNING]
> Если в вашей анимации есть два соседних ключ-кадра, и у первого из них
> имеются трансформации кости с одним из следующих
> атрибутов: `out-interp`; `out-rotation-interp`; `interp`; `rotation-interp`,
> и при этом у второго ключ-кадра имеются трансформации кости с одним из
> следующих атрибутов: `in-interp`; `in-rotation-interp`; `interp`; `rotation-interp`,
> то приоритет будет отдан интерполяции из второго ключ-кадра.

***Допустимые вложенные элементы***

`position`;\
`rotation`;\
`scale `.

***Пример***

```
... {
    ... {
        ...
        
        @bone name "spine" {
            ...
        }
        
        @bone name "head" {
            ...
        }
    }
}
```

### Элемент `union`

***Описание***

Используется для обобщения интерполяционных параметров костей.

***Атрибуты***

- `interp`, `rotation-interp`, `in-interp`, `out-interp`, `in-rotation-interp`, `out-rotation-interp`.

***Допустимые вложенные элементы***

`union`;\
`bone `.

***Пример***

```
... {
    ... {
        @union interp "lerp" {
            ...
            
            @union interp "cubic-spline" rotation-interp "slerp" {
                ...
            }
        }
    }
}
```

### Элемент `position`

***Атрибуты***

- `value` - массив вида `(x, y, z)`, представляющий позицию;
- `interp`, `in-interp`, `out-interp`.

***Пример***

```
... {
    ... {
        ... {
            @position value (0, 0, 0)
        }
    
        ... {
            @position value (0, 2, 2) interp "lerp"
        }
        
        ... {
            @position value (0, 4, 4) interp "cubic-spline"
        }
    }
}
```

### Элемент `rotation`

***Атрибуты***

- `euler` - массив вида `(x, y, z)`, представляющий
вращение углами Эйлера в градусах;
- `quat` - массив вида `(x, y, z, w)`, представляющий
вращение кватернионом;
- `rotation-interp`, `in-rotation-interp`, `out-rotation-interp`.

> [!NOTE]
> `euler` и `quat` не могут быть определены вместе. Используйте один из вариантов.

***Пример***

```
... {
    ... rotation-interp "slerp" {
        ... {
            @rotation euler (0, 0, 0)
        }
    
        ... {
            @rotation euler (90, 90, 90) rotation-interp "nlerp"
        }
    }
}
```

### Элемент `scale`

***Атрибуты***

- `value` - массив вида `(x, y, z)`, представляющий масштаб;
- `interp`, `in-interp`, `out-interp`.

***Пример***

```
... {
    ... {
        ... {
            @scale value (0, 0, 0)
        }
    
        ... {
            @scale value (0, 2, 2) interp "lerp"
        }
    }
}
```

## Элементы для сложной анимации

### Элементы `interp`

***Описание***

Используются для конкретизации параметров для некоторых видов интерполяций (в т.ч. `cubic-spline`)

***Атрибуты***

- `id` - идентификатор конкретизированной интерполяции, который можно будет использовать в анимациях;
- `type` - тип конкретизируемой интерполяции. Доступные значения: `cubic-spline`.

***Поля***

Для описания параметров различных видов интерполяций, используется под-элемент `field`,
имеющий атрибуты `name` и `value`, которые содержат имя и значение параметра.

**Поля для `cubic-spline`**

- `in-tangent` - вектор, описывающий направление и скорость изменения значения при входе в ключ-кадр.
- `out-tangent` - вектор, описывающий направление и скорость изменения значения при выходе из ключ-кадра.

***Пример***

```
@interp id "custom-interp" type "cubic-spline" {
    @field name "in-tangent" value (0,3,1)
    @field name "out-tangent" value (3,3,1)
}
```

**Поля для `squad`**

- `in-control` - вспомогательный кватернион, определяющий кривизну траектории при входе в данную точку. 
Влияет на форму сегмента перед этим ключ-кадром;

- `out-control` - вспомогательный кватернион, определяющий кривизну траектории при выходе из данной точки. 
Влияет на форму сегмента после этого ключ-кадра.

***Пример***

```
@interp id "custom-interp" type "squad" {
    @field name "in-control" value (0,1,0,1)
    @field name "out-control" value (1,0.32,1,0.8)
}
```
